# Narrative Architect Skill

[English](README.md) | **简体中文**

把抽象、技术或敏感的内容，变成清晰、贴合受众的表达。

大多数稿子都以同一种安静的方式失败：事实都对，但没人感受到它为什么重要。

Narrative Architect Skill 帮你在复杂的素材里找到压力、冲突和转折，再为你真正需要打动的那个人打磨表达。它不会让所有人都喜欢你的话。它让对的听众理解这件事为什么重要。

这个 skill 在保留原意和事实边界的前提下，提升叙事结构、情绪力度、受众贴合度与表达的"不像 AI 写"的自然度。

它受故事原理方法（包括 Robert McKee 风格的叙事思路）启发，但与 Robert McKee 或任何第三方写作项目没有隶属关系。

## 它能做什么

- **找到叙事内核** —— 什么处在风险中，哪个旧假设正在被打破
- **读懂受众** —— 他们知道什么、害怕什么、从哪里切入
- **控制节奏** —— 什么时候让读者安心，什么时候给他们惊喜
- **让事实判断停留在已知边界之内**

对于有素材依据的工作，事实判断始终扣住已知材料。如果素材有缺口，输出会把缺口标出来，而不是用猜测去填上。

## 常见任务

在用户提出以下需求时使用：

- 重写一封邮件、消息、帖子、提案、README、文章、演讲稿或路演稿
- 把技术材料翻译成非技术解释
- 起草面向创始人、投资人、客户、合作伙伴或高管的沟通
- 改善一份听起来太通用、太 AI、太正式、太销售味或太抽象的稿子
- 把同一个想法改写成不同受众（开发者、投资人、客户、高管或大众读者）能各自接收的版本
- 在保留事实判断的前提下，把沟通改得更锐利、更自然、更有说服力

## 示例 Prompt

- "把这份创始人周报重写一版，更锐利、别那么通用。"
- "把这个技术概念讲清楚，给没有工程背景的高管听。"
- "帮我写一个开源项目的发布贴。"
- "重写这个 README 的开头，让开发者更快 get 到价值。"
- "把产品介绍改写成投资人能听懂的叙事。"
- "让这条消息更暖、更直接、更不像 AI 写的。"
- "把这个抽象的想法变成一个别人记得住的故事。"

## 适合谁

写作者、创始人、开发者、市场人员、运营和产品团队用这个 skill 起草和重写高 stakes 的沟通：邮件、路演稿、发布贴、演讲、README 开头、提案、个人消息、高管摘要，以及技术到非技术的转译。

素材偏抽象、技术、敏感、战略或太通用时，这个 skill 特别有用。它会提升叙事结构、受众贴合度、事实保真度、情绪力度和自然度，同时避免过度推销或没有依据的判断。

## 它怎么工作

这个 skill 会选择能完成任务的最轻量模式。短消息不会被吹成提案。一次纠正不会只做表面润色。

| 模式 | 什么时候用 |
| --- | --- |
| Quick Message（短消息） | 简短留言、微信、跟进 |
| One-on-One Strategy（一对一策略） | 联系某个人、团队、投资人、专家或合作伙伴 |
| Public Narrative（公开叙事） | 文章、演讲、品牌故事、社交平台帖子 |
| Expert Translation（专家转译） | 把一个概念讲给另一类受众 |
| Rewrite Recovery（重写修复） | 修复一份没打动人的稿子 |
| Personal Communication（个人沟通） | 个人、情感、友谊或敏感话题的消息 |
| Full Narrative（完整叙事） | 重大路演、宣言、主题演讲、长文 |

## 原则

**讲故事，而不是推销。**
目标不是让所有人喜欢。是与对的听众产生强共鸣。

**有控制的意外。**
好的沟通在"我懂了"和"我没从这个角度看过"之间来回移动。太可预测会无聊，太混乱会读不下去。这个 skill 在管理这种节奏。

**受众优先，权威保留。**
受众决定切入点与节奏。你的中心判断不变。

**事实边界。**
对于有素材依据的工作，事实判断始终扣住已知材料。如果素材有缺口，输出会把缺口标出来，而不是用猜测去填上。

## 安装

克隆仓库：

```bash
git clone https://github.com/gptmaas/Narrative-Architect-Skill.git
```

常见的本地做法，是把 skill 目录放在你 OpenClaw 工作区的 `skills/` 下：

```
skills/narrative_architect_skill/
```

具体依赖取决于你的 OpenClaw 运行时与模型提供方配置。

## 项目结构

仓库的简化结构如下：

```
Narrative-Architect-Skill/
├── LICENSE
└── narrative_architect_skill/
    ├── skill.md              # 规则与原则
    ├── runtime_chain.md      # 运行链与模式
    ├── mappers/              # 故事结构、受众、信息熵
    ├── principles/           # 核心叙事原则
    ├── selectors/            # 运行模式与切入点选择
    ├── renderers/            # 渲染器与风格封装
    ├── harness/              # 质量与校验检查
    ├── contracts/            # 输出格式契约
    ├── policies/             # 行为策略
    └── cases/                # 回归测试
```
