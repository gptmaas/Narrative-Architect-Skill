# Non-Offensive Personality Harness

## Purpose

Non-Offensive Personality Harness ensures that the skill preserves user personality, judgment, and tension without creating unnecessary offense or hostility.

the skill should not make every message bland.
But it must not turn personality into arrogance or tension into attack.

## What It Checks

1. Does the output show personality?
2. Does the output preserve the user's judgment?
3. Does it have tension?
4. Does it avoid unnecessary offense?
5. Does it criticize structure rather than humiliating people?
6. Does it avoid sounding superior?
7. Does it allow the wrong listener to be neutral rather than offended?

## Personality Signals

Healthy signals:

```text
clear judgment
specific worldview
sharp distinction
controlled confidence
taste
restraint
urgency
intellectual honesty
```

Unhealthy signals:

```text
contempt
mockery
self-importance
superiority
dismissiveness
moral pressure
exaggerated certainty
```

## Tension Without Hostility

Good tension:

```text
The old way fails because it makes risk visible too late.
```

Bad hostility:

```text
People who still use the old way do not understand the future.
```

Good tension:

```text
This is not a product demo; it is a real-world testbed for a hard method problem.
```

Bad hostility:

```text
Most people are still doing shallow product demos.
```

## Checkpoints

### 1. Personality Present

Fail if the output is so polite that it becomes generic.

### 2. No Contempt

Fail if the tone suggests the listener or industry is foolish.

### 3. Structural Critique

Prefer criticism of systems, incentives, assumptions, or methods.

Avoid criticism of people.

### 4. Selective Appeal

It is acceptable if the text does not appeal to everyone.

It is not acceptable if it unnecessarily antagonizes non-target listeners.

### 5. Evidence-Bound Confidence

Confidence must come from evidence, experience, or clear reasoning.

Do not use confidence as performance.

## Failure Modes

Hard fail:

1. The output insults, mocks, or humiliates.
2. The output implies disagreement equals ignorance.
3. The output creates unnecessary enemies.
4. The user's personality becomes aggression.

Soft fail:

1. The output is too bland.
2. The output hides all judgment.
3. The tension is too weak.
4. The message tries too hard to be liked.

## Rewrite Rule

If the output is too bland:

```text
restore the user's judgment
name the old-world assumption
make the contrast sharper
keep the tone respectful
```

If the output is too offensive:

```text
remove personal judgment
criticize structure, not people
replace contempt with clarity
reduce exaggerated certainty
```

## Final Rule

the skill should help the user be vivid, not hostile.

The right listener should feel invited into a sharper story, not attacked by it.

## High-Status Recipient Tone

For high-status or high-power recipients, non-offense requires extra discipline.

Avoid:

```text
telling them what they fail to understand
diagnosing their organization too directly
sounding like a fan
sounding like a critic
sounding like a consultant
using superiority as tone
```

Prefer:

```text
naming a structural problem
offering a narrow test
respecting time
keeping the ask small
letting the proof speak
```

## Rewrite Rule

If the tone sounds like fan mail, criticism, or consulting diagnosis, rewrite toward:

```text
structural problem
→ narrow capability
→ small proof challenge
```
