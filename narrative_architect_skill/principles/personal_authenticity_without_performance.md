# Personal Authenticity Without Performance

## Purpose

This principle governs personal and romantic communication.

It helps the user communicate honestly without self-pity, over-performance, or manipulative framing.

## Core Principle

```text
Do not sell yourself.
Create a low-pressure situation where the real version of you can show up.
```

中文：

```text
不要把自己卖出去。
创造一个低压力场景，让真实的你自然出现。
```

## What This Means

A user may describe themselves as:

```text
awkward
geeky
too smart
not smooth
bad at talking
shy
overthinking
```

the skill should not turn these into defects.

It should reframe them as known conditions that can be handled with better context.

## Good Reframes

Instead of:

```text
I am bad at talking.
```

Use:

```text
I am better when we are doing something than when I am trying to act smooth.
```

Instead of:

```text
I am awkward.
```

Use:

```text
I may be a little slow at first, but I am much more natural when there is something real to talk about.
```

Instead of:

```text
I am super smart.
```

Use:

```text
I care about things deeply, and sometimes I get more thoughtful than smooth.
```

## Scene Over Performance Rule

For users who struggle with small talk, recommend a setting that reduces performance pressure.

Good settings:

```text
bookstore
museum
quiet walk
coffee with an activity nearby
board game
strategy game
small exhibit
workshop
maker space
specialty shop
place the user genuinely knows and likes
```

Bad settings:

```text
high-pressure dinner
loud bar
standing small talk
overly romantic setup
expensive performance date
```

## Message Rules

A good personal message should be:

```text
short
honest
specific
low-pressure
easy to accept or decline
```

It should not:

```text
beg
apologize for existing
over-explain insecurity
sound like a script
make the other person responsible for the user's confidence
```

## Failure Modes

Hard fail:

1. Makes the user sound needy.
2. Makes the user sound arrogant.
3. Uses pickup tactics.
4. Pretends the user is smooth.
5. Overstates vulnerability too early.
6. Makes the recipient responsible for rescuing the user.
7. Ignores the user's real personality.

Soft fail:

1. Good advice but message too long.
2. Message too polished.
3. Message too abstract.
4. Too much reassurance.
5. Too much self-labeling.

## Final Rule

The best personal message does not hide awkwardness.

It gives awkwardness a safe shape.
