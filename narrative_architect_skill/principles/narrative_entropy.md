# Narrative Entropy Principle

## Purpose

Narrative Entropy Principle defines how Robert Skill manages predictability, surprise, cognitive load, and meaning.

Information entropy, in the narrative sense, refers to how predictable or unpredictable the next piece of information feels to the audience.

A story with too little entropy feels boring. 
A story with too much entropy feels chaotic. 
A good story moves between familiarity and surprise.

## Core Principle

```text
Low entropy gives the audience ground.
High entropy gives the audience discovery.
Great narrative alternates between the two.
```

中文：

```text
低熵让听众站稳，高熵让听众被击中。好的叙事在两者之间摆动。
```

## What Low Entropy Does

Low-entropy information is familiar, predictable, easy to process.

Use low entropy to:

```text
establish context
define the scene
make the audience feel safe
explain necessary background
create rhythm
lower cognitive load
```

中文表达：

```text
低熵信息负责铺地板：让听众知道自己站在哪里。
```

Examples:

```text
一家大公司的数据通常不是没有，而是散在很多系统里。
AI 可以写出很顺的答案。
一个企业出海前，需要知道法规、医院、渠道和支付规则。
```

## What High Entropy Does

High-entropy information is unexpected, meaning-changing, or assumption-breaking.

Use high entropy to:

```text
create tension
break an old assumption
reveal hidden risk
introduce a new frame
produce emotional recognition
make the audience lean in
```

中文表达：

```text
高熵信息负责打断自动驾驶：让听众意识到，原来的理解不够用了。
```

Examples:

```text
最麻烦的不是 AI 写得差，而是它写得足够好，好到你看不出错。
真正被压缩的不是时间，而是盲撞的次数。
问题不在数据缺失，而在数据之间断了线。
```

## Entropy Rhythm

Robert Skill should create an entropy rhythm:

```text
familiar ground
→ small surprise
→ explanation
→ larger reversal
→ resolution
```

中文：

```text
熟悉场景
→ 轻微意外
→ 解释机制
→ 更深反转
→ 价值收束
```

## Entropy Failure Modes

### Too Low Entropy

The output is clear but boring.

Symptoms:

```text
definition-heavy
predictable
safe
polite but forgettable
no reversal
no tension
no new frame
```

### Too High Entropy

The output is exciting but confusing.

Symptoms:

```text
too many metaphors
too many jumps
too many new concepts
unclear protagonist
unexplained claims
emotional overdrive
conceptual overload
```

## Entropy Rule by Audience

### General Audience

Use low-entropy entry, then high-entropy insight.

Pattern:

```text
familiar scene → hidden problem → simple mechanism → emotional resolution
```

### Technical Audience

Use medium entropy. Avoid theatrical surprise. Use precise contrast.

Pattern:

```text
known limitation → system-level constraint → control mechanism → engineering consequence
```

### Investor / Strategic Audience

Use high-value reframing.

Pattern:

```text
expensive failure → misunderstood market → new system logic → scale implication
```

### One-on-One Communication

Use controlled entropy. The listener should feel the message is relevant, not ambushed.

Pattern:

```text
relationship context → shared problem → unexpected overlap → small next step
```

### Child Audience

Use low entropy and one surprise only.

Pattern:

```text
familiar game/map → hidden trap → helper tool → safer next move
```

## Final Rule

A Robert Skill output should not be only understandable.

It should create at least one meaningful shift:

```text
I thought it was X.
Now I see it is really Y.
```
